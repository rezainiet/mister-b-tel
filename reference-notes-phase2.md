# Notes de référence — Mister B

## PDF dashboard Prestige

Source : `/home/ubuntu/upload/GuidecompletReproduireleDashboardPrestigesurunautrebusiness.pdf`

Le PDF n’est pas une simple charte graphique : il documente aussi l’architecture technique du dashboard. En revanche, il confirme plusieurs éléments de structure à reproduire dans l’interface finale du dashboard Mister B.

La direction visuelle observée est **sobre, premium et très lisible**. Les pages montrent un fond clair, de grands blocs blancs arrondis, une hiérarchie très marquée par de gros titres, des sous-titres plus discrets, et des tableaux/cartes avec coins généreusement arrondis. La composition repose sur des sections larges, aérées, avec beaucoup d’espace négatif.

Les indices explicites utiles pour le dashboard sont les suivants :

| Élément | Observation utile |
|---|---|
| Architecture | Le dashboard est présenté comme une couche distincte du tracking client et de l’API serveur. |
| Rôle dashboard | Le client dashboard « affiche les stats en temps réel avec graphiques et feed live ». |
| Stack | Recharts est explicitement indiqué pour les graphiques. |
| Style visuel | Cartes et blocs clairs, très arrondis, fond blanc cassé, bordures fines et ombres légères. |
| Lisibilité | Titres lourds, sections bien séparées, peu d’ornements, approche très éditoriale. |

Ces notes confirment que la refonte doit privilégier une **grande lisibilité**, un **header structuré**, des **cartes métriques imposantes**, un **graphique principal**, puis des blocs secondaires de détail.
